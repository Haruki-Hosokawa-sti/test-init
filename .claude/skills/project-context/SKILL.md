# プロジェクトコンテキスト

## プロジェクト概要
- **このプロジェクトが何をするか**: 技術スタックの学習・検証を目的としたテストプロジェクト
- **ドメイン**: 開発者ツール

## ドメイン制約
特になし（学習/テスト目的のため、厳格なドメイン制約は不要）

## 開発フェーズ
- **フェーズ**: プロトタイプ（概念実証・初期検証段階）

## ディレクトリ規約
モノレポ標準構造に従う:
- `packages/front/` - Next.js フロントエンド（App Router）
- `packages/back/` - NestJS バックエンド（GraphQL + Prisma）

## Git WorkTree 運用
- 新規要件は `feature/<タスク概要>` ブランチで作業
- `git worktree add` でWorktreeを作成し、作業完了後にPR作成

## Docker Compose 構成
- サービス: front (3000), back (4000), db (5432)
- ポート競合チェック → 自動停止（同一プロジェクトのみ）→ 起動
